# DX i18n Achievements

`i18n` is the Rust internationalization workspace for DX. It combines
translation provider boundaries, local-first localization, optional
Lingo.dev-compatible execution, text-to-speech, speech-to-text, and
Wisprflow-style examples behind one crate.

## Position

i18n should be the language layer for DX: local catalogs first, provider calls
only when explicitly requested, protected source structure, auditable lockfiles,
and command boundaries that can verify locale state without cloud credentials.

Professional positioning:

> i18n is a Rust localization crate with auth-free local status, lockfile, and
> run workflows; JSON and Markdown bucket handling; protected placeholders and
> Markdown/code structure; optional Lingo.dev-compatible HTTP execution; and
> separate TTS/STT source modules.

That statement avoids the older embedded-model and speed wording in the README. In
this checkout, `models/ggml-tiny.en.bin` is not present, so embedded Whisper
model availability and CPU transcription speed need fresh runtime verification before
being repeated.

## What i18n Has Already Achieved

| Area | Achievement |
| --- | --- |
| Rust crate surface | Exposes `locale`, `localization`, `sts`, `tts`, and optional `wisprflow` modules from `src/lib.rs`. |
| CLI boundary | Provides the `dx-i18n` binary for `show config`, `status`, `lockfile`, and `run` workflows over a configured workspace root. |
| Local-first workflow | Keeps `status`, `lockfile`, `lockfile --check`, and default `run` auth-free for local JSON and Markdown localization. |
| Lingo-compatible config | Parses Lingo.dev-style `i18n.json` shape, source/target locales, buckets, include/exclude rules, `engineId`, raw provider config, and key controls. |
| Lockfile model | Stores and compares source and key fingerprints so current, changed, new, and renamed strings can be reasoned about without byte-for-byte YAML dependence. |
| Workspace filters | Supports scoped locale, bucket, file, and key filters while rejecting empty selections before writes. |
| Frozen verification | Implements `run --frozen` as a no-write gate for selected source lock state and rendered target drift. |
| Protected text | Protects placeholders, double-brace interpolation, ICU-style variables, inline code, fenced code blocks, Markdown links/emphasis/tables, and HTML-like structures. |
| Safe local rendering | Reuses existing target JSON and Markdown only when protected tokens and structural shape stay compatible; otherwise it falls back to source text instead of guessing. |
| Lingo HTTP adapter | Sends API keys in headers, validates response locale/data shape, keeps keys out of request JSON, bounds successful/error bodies, redacts error text, disables redirects, and validates custom API URLs. |
| Explicit raw-provider boundary | Recognizes raw provider IDs for compatibility but fails direct raw-provider execution before writing because those adapters are not implemented yet. |
| TTS/STT source modules | Keeps Microsoft Edge TTS, Google TTS/STT, Whisper STT, cached Whisper, and auto STT source paths available behind the crate feature model. |
| Test footprint | Contains focused Rust tests for localization workflow, CLI behavior, Lingo-compatible fixtures, workspace status, protected text, lockfile, config, and Markdown/JSON handling. |
| Disabled CI policy | Keeps workflow files disabled while the current operating rule is to keep GitHub Actions paused. |

## Public Positioning Boundary

Safe internal statement:

> i18n owns a local-first Rust localization workflow with JSON/Markdown
> document handling, protected structure, lockfile fingerprints, scoped
> verification, optional Lingo.dev-compatible HTTP execution, and TTS/STT source
> modules.

Safe public wording after refreshed checks:

> The current tests prove the selected local-first localization, CLI filter,
> no-write status, frozen-run, protected-token, and Lingo-compatible adapter
> behavior covered by the checked test suite.

Unsafe statement without more verification:

> i18n ships an embedded Whisper model in this checkout, proves the README's
> CPU transcription speed, has live Google/DeepL/Lingo provider availability,
> implements direct raw providers, or is release-qualified for every STT/TTS path.

The strongest current result is the safety and ergonomics of the localization
boundary. The remaining verification work is refreshing the test run, reconciling the
missing embedded STT model path, and separating provider/network runtime verification
from repository-level adapter code.
